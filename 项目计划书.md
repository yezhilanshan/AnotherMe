项目名称：AnotherMe 智能多智能体互动课堂与题目讲解视频生成平台

一、项目概述

本项目面向“AI+教育”场景，拟建设一套能够将任意主题、课程要求、PDF 学习资料乃至题目图片，自动转化为沉浸式互动课堂与讲题视频的智能教学平台。项目依托现有 AnotherMe 原型系统与 AnotherMe2 多智能体引擎，形成“课堂生成 + 多智能体授课 + 白板讲解 + 交互测验 + 项目式学习 + 题目视频生成”的一体化能力链条。

平台前端基于 Next.js 与 React 构建，支持课堂创建、生成预览、课堂回放、互动学习、题目视频等功能；后端通过 API 调度课堂生成、媒体生成、课堂存储、语音合成和搜索增强；底层算法引擎基于 LangGraph 和多智能体协同机制，完成视觉理解、学情建模、脚本生成、语音生成、动画编排、错误修复和音视频合成。项目的目标不是简单提供一个聊天机器人，而是构建一个具备“课程设计能力、课堂组织能力、讲解表达能力和资源生成能力”的智能教学系统。

二、立项背景与项目意义

当前数字教育资源建设面临三类突出问题。第一，优质课程内容生产成本高，教师准备一节结构完整、形式丰富、兼顾互动与练习的课程需要投入大量时间。第二，现有大模型教育应用多停留在问答层面，缺少完整的课堂组织和教学编排能力，难以形成连续、沉浸、可回放的学习体验。第三，题目讲解与个性化辅导往往碎片化，缺少基于学情分析和多模态表达的精准支持。

本项目的意义主要体现在以下几个方面。

1. 提升教学资源生产效率。教师或学生只需输入主题、上传资料或提供题目，即可自动生成课程大纲、课堂场景、测验题、交互模块和讲题视频，大幅缩短内容准备周期。

2. 推动课堂形态升级。系统不再局限于单一页面展示，而是形成“AI 教师 + AI 同学 + 白板 + 语音 + 幻灯片 + 交互任务”的复合教学形态，增强课堂临场感和参与感。

3. 支撑个性化学习。通过学情建模、知识点识别、错因分析和自适应策略分发，平台能够面向不同学习基础提供差异化讲解路径。

4. 拓展教育普惠边界。平台支持本地部署、云端部署和轻量化接入，适用于学校课堂、课后辅导、家庭学习和在线教育平台等多种场景，有利于提升优质教育资源的可达性。

三、项目建设目标

本项目拟围绕“自动生成、智能互动、个性辅导、可持续扩展”四个维度开展建设，形成可演示、可部署、可推广的智能课堂平台。

总体目标是建设一套具备以下能力的教育智能体系统：

1. 支持基于文本主题、课程需求和 PDF 资料的一键式课堂生成。
2. 支持幻灯片、测验、交互式 HTML 模块、PBL 项目制任务等多种课堂场景自动编排。
3. 支持多智能体角色生成与课堂协作，形成教师、助教、学生等多角色互动机制。
4. 支持白板书写、语音讲解、媒体生成与课堂回放，构建沉浸式学习体验。
5. 支持题目图片上传与自动讲题视频生成，实现从题目识别到讲解视频输出的闭环。
6. 支持课堂内容持久化、任务异步调度、断点恢复与后续二次编辑。
7. 形成面向教育场景的技术架构、原型平台、示范案例与可推广实施方案。

四、项目现有基础

从现有项目代码与系统结构看，项目已具备较好的技术原型基础，不属于从零起步，而是具备进一步申报和深化建设的条件。

1. 已形成完整的前端产品形态。系统包含互动课堂工作台、课堂生成预览页、课堂回放页、题目视频页、设置页等核心页面，具备较完整的用户操作链路。

2. 已建立课堂数据模型。平台以 `Stage + Scenes` 为核心组织课堂数据，场景类型覆盖 `slide`、`quiz`、`interactive`、`pbl` 四类，能够承载多元教学活动。

3. 已实现课堂生成 API 机制。系统支持异步创建课堂生成任务、轮询进度、完成后自动持久化课堂数据，并支持后续课堂加载与恢复。

4. 已具备多智能体角色生成能力。系统可以根据课程主题和场景大纲自动生成教师、助教、学生等角色的名称、人格、头像、颜色与优先级配置。

5. 已具备题目讲解视频生成引擎。AnotherMe2 引擎采用多智能体工作流，能够完成题目图像理解、学情建模、分步脚本生成、语音生成、动画生成、错误修复和最终视频输出。

6. 已具备跨模态扩展能力。系统支持接入大模型服务、联网搜索、TTS、图像与视频生成能力，并为后续扩展更多服务商和教学模块预留接口。

五、项目建设内容

1. 智能课堂自动生成模块

系统根据用户输入的学习主题、课程描述或 PDF 教材内容，自动生成课程大纲，并进一步扩展为多个结构化场景。每个场景可以是知识讲解型幻灯片、课堂测验、互动模拟实验或项目式学习任务。该模块解决“从需求到课堂内容”的自动化问题，是平台的核心入口。

2. 多智能体课堂组织模块

系统将传统单一 AI 助手升级为课堂组织型多智能体系统。平台可自动生成教师、助教和学生角色，使不同角色在课堂中承担不同任务，如主讲、提问、追问、讨论、总结与反馈等，从而增强课堂的参与性和真实感。

3. 多模态教学表达模块

平台支持语音讲解、白板绘制、图文展示、互动页面和项目任务等多种教学表达方式。相较于传统的纯文本回答或静态课件，本项目更加重视“讲解过程”的可视化和动态化，使抽象知识能够以更直观的方式传达给学习者。

4. 学情建模与自适应教学模块

在题目讲解视频场景中，系统已引入学情建模模块，对知识点前置关系、掌握度、常见错误和解释策略进行建模，并据此生成更适合学习者的讲解路径。后续可进一步将该能力推广到课堂生成过程，实现针对不同年级、不同能力学生的差异化内容生成。

5. 题目讲解视频生成模块

系统支持上传题目图片，由视觉智能体识别题目内容与几何关系，脚本智能体生成分步讲解文本，语音智能体生成配音，动画智能体生成 Manim 动画代码，修复智能体自动处理常见渲染错误，最终生成可播放的讲题视频。该模块特别适合数学、几何、物理等强调步骤推导的学科。

6. 课堂存储与任务调度模块

平台支持课堂数据持久化、任务状态管理、异步后台执行、生成进度查询和失败恢复。系统可在课堂切换、页面刷新或任务未完成时自动恢复状态，保障用户体验和系统稳定性。

六、技术路线与系统架构

本项目总体采用“前端交互层 + 服务编排层 + 多智能体引擎层 + 资源生成层 + 存储与运维层”的分层架构。

1. 前端交互层

基于 Next.js 16、React 19 与 TypeScript 构建用户界面，负责主题输入、资料上传、生成预览、课堂播放、结果展示与交互操作。该层强调课堂产品化体验，支持课程创建、播放回放、媒体加载和用户状态恢复。

2. 服务编排层

基于 Next.js API Routes 提供统一后端接口，负责课堂生成任务创建、课堂查询、题目视频任务创建、模型调用路由、课堂媒体生成、课堂数据持久化和健康检查。该层承担业务编排与系统集成职责。

3. 多智能体引擎层

底层引擎基于 LangGraph 组织多智能体工作流。在题目视频场景中，已形成“Vision → Learner Modeling → Script → Voice → Animation → Repair → Merge”的链式流程。该流程体现出明确的任务分工和状态传递机制，适合复杂教育内容生成场景。

4. 资源生成层

平台整合大模型推理、语音合成、图像生成、视频生成、网页交互生成和白板渲染等能力，实现多模态教学资源自动生产。课堂不仅包含文字和图片，还可以包含语音、视频、互动模拟和项目任务。

5. 存储与运维层

系统采用文件化课堂存储与任务存储机制，支持课堂 JSON 持久化、任务状态追踪、原子化写入与恢复加载。部署层支持本地运行、Docker 部署和云端部署，具备较好的可推广性。

七、关键技术问题

1. 从自然语言需求到结构化课堂的自动转化问题。需要解决课程目标抽取、知识点拆分、场景排序和教学活动编排问题。

2. 多智能体协同教学的问题。不同角色之间需要形成清晰的分工、时序与话语风格控制，避免输出冗余和角色混乱。

3. 多模态内容生成的一致性问题。文字讲解、幻灯片内容、语音脚本、白板动作和动画表现之间必须保持语义一致和节奏统一。

4. 学情建模与个性化输出问题。系统需要把知识图谱、掌握度评估、常见错误和解释策略映射到具体教学内容生成中。

5. 自动动画生成与修复问题。题目视频场景中，动画代码生成容易出现渲染错误，因此需要结合模板化生成、布局管理与规则修复机制提升稳定性。

6. 生成任务稳定调度问题。教育内容生成往往耗时较长，需要任务异步执行、进度可视化、失败可追踪和状态可恢复。

八、项目创新点

1. 从“对话问答”升级为“课堂生成”。本项目不是简单让模型回答问题，而是让模型完成课程设计、场景组织、角色分配、活动编排和内容生成，具有更强的教育产品属性。

2. 多智能体驱动课堂互动。平台引入教师、助教、学生等多类角色，使课堂具备讨论、追问、点评和协作特征，突破传统单智能体教学模式。

3. 统一支持四类课堂场景。系统在同一平台内打通幻灯片、测验、交互页面和 PBL 任务，形成兼具讲授、练习、探究和实践的复合式课堂结构。

4. 学情建模嵌入内容生成流程。题目视频引擎已引入知识点、前置关系、掌握度和错误类型分析，使讲解不再是“统一答案输出”，而是向“因材施教”演进。

5. 形成“题目图片到讲题视频”的完整闭环。系统将 OCR、几何理解、脚本生成、语音合成、动画编排与视频输出连接起来，体现出较强的教育垂类深度。

6. 具备自动修复与工程化落地能力。系统不仅能生成内容，还能通过规则修复、任务持久化和断点恢复增强真实可用性，体现出从研究原型向应用系统转化的特点。

九、应用场景

1. 中小学课堂教学。教师可快速生成导入课、复习课、专题训练课和实验探究课。

2. 课后辅导与自主学习。学生可围绕知识点、错题或考试专题生成专属课堂和讲题视频。

3. 教师备课与资源共享。教师可利用平台快速形成初始课件、测验与互动任务，再进行二次编辑和共享。

4. 在线教育平台。平台可作为课程内容自动化生产工具，提高内容更新效率。

5. 教育机构和培训场景。可用于专题班、竞赛辅导、学科诊断和项目式学习组织。

十三、风险分析与应对措施

1. 生成内容准确性风险。对策是引入知识点校验、模板约束、教师二次审核和示范案例库，降低幻觉与知识错误。

2. 多模型接口稳定性风险。对策是保留多服务商适配能力，建立统一模型解析、配置和降级机制。

3. 任务耗时与成本风险。对策是采用异步任务、分阶段生成、可选联网搜索和媒体生成开关，控制资源消耗。

4. 个别场景渲染失败风险。对策是继续完善规则修复、布局管理和失败重试机制，提升动画生成成功率。

5. 数据与隐私风险。对策是加强本地部署支持、最小化数据采集、做好题目图片和课堂数据的权限管理。
