可以总结为一句话：

> **学生 Agent 负责“刻画学生现在会什么、不会什么”；导师 Agent 负责“基于这个状态决定怎么教、教什么、讲到什么程度”。**

下面按“定位—能力—输入输出—实现步骤”给你整理。

---

# 一、学生 Agent 的实现总结

## 1. 定位

学生 Agent 不是聊天机器人，本质上是一个 **学生状态建模器**。

它要回答三个问题：

* 这个学生当前掌握了哪些知识点
* 这个学生在哪类能力上强/弱
* 面对一道题时，他大概率会卡在哪一步

---

## 2. 核心建模内容

建议建三类状态：

### （1）知识点掌握度

例如：

* 相似三角形
* 圆幂定理
* 勾股定理
* 垂径定理

表示方式：

```json
{
  "similar_triangle": 0.78,
  "circle_power": 0.42,
  "pythagorean": 0.81
}
```

---

### （2）通用能力向量

例如：

* 几何建模能力
* 辅助线构造能力
* 多步推理能力
* 代数转化能力

表示方式：

```json
{
  "modeling": 0.63,
  "auxiliary_line": 0.37,
  "multi_step_reasoning": 0.51
}
```

---

### （3）行为特征

例如：

* 粗心率
* 作答速度
* 偏好图形法还是代数法
* 遇难题是否容易中断

表示方式：

```json
{
  "careless_rate": 0.18,
  "response_speed": 0.66,
  "geometry_preference": 0.72
}
```

---

## 3. 学生 Agent 的输入

主要输入有四类：

* 学生历史做题记录
* 学生提问记录
* 题目的知识点标签
* 题目的标准解题步骤

---

## 4. 学生 Agent 的输出

输出不要写成自然语言为主，而要尽量结构化：

```json
{
  "mastery": {
    "similar_triangle": 0.78,
    "circle_power": 0.42
  },
  "skills": {
    "auxiliary_line": 0.37,
    "multi_step_reasoning": 0.51
  },
  "current_risk_step": 2,
  "likely_failure_reason": "auxiliary_line_weak"
}
```

---

## 5. 学生 Agent 的实现机制

核心流程是：

### 第一步：给题目标注知识点和能力需求

例如一道题需要：

* 相似三角形
* 辅助线
* 两步推理

---

### 第二步：把标准解拆成步骤

例如：

```json
[
  {"step": 1, "skill": "angle_relation"},
  {"step": 2, "skill": "auxiliary_line"},
  {"step": 3, "skill": "similar_triangle"},
  {"step": 4, "skill": "multi_step_reasoning"}
]
```

---

### 第三步：结合学生模型做逐步通过预测

例如第 2 步需要辅助线能力，而该学生辅助线能力较弱，就判断这里高概率卡住。

可以用简单打分：

```python
prob = 0.6 * knowledge_score + 0.4 * skill_score
```

也可以后期升级成：

* 贝叶斯知识追踪
* IRT
* 深度知识追踪

---

## 6. 学生 Agent 的本质

本质上它做两件事：

* **静态画像**：学生整体会什么
* **动态预测**：碰到当前这道题会卡哪

---

# 二、导师 Agent 的实现总结

## 1. 定位

导师 Agent 不是单纯答疑模型，而是一个 **教学决策器 + 讲解生成器**。

它要回答三个问题：

* 学生当前的问题是什么
* 应该采用哪种教学策略
* 应该生成什么样的讲解或追问

---

## 2. 导师 Agent 的核心能力

### （1）理解学生问题

识别学生是在问：

* 整题不会
* 某一步为什么这样做
* 辅助线怎么想到
* 自己答案哪里错了

输出类似：

```json
{
  "intent": "ask_why_step",
  "target_step": 2
}
```

---

### （2）读取学生状态

调用学生 Agent 的结果，知道：

* 他是否缺知识点
* 还是缺方法
* 还是只是粗心

---

### （3）选择教学策略

常见策略至少做 4 类：

* 直接讲解
* 提示式讲解
* 引导式提问
* 纠错式讲解

例如：

* 学生很弱：直接讲
* 学生中等：引导讲
* 学生较强：只给提示
* 学生做错：指出错误根因

---

### （4）生成个性化讲解

同一题对不同学生输出不同：

* 弱学生：拆细讲
* 强学生：点关键
* 卡在辅助线：重点讲“为什么要连这条线”
* 卡在推理：重点讲步骤衔接

---

### （5）维护上下文记忆

记录：

* 这道题已经讲过哪些部分
* 学生连续追问的焦点是什么
* 学生当前仍没理解的点是什么

---

## 3. 导师 Agent 的输入

主要输入有：

* 学生当前提问
* 学生 Agent 画像结果
* 当前题目及解题步骤
* 历史对话上下文
* 教学模式（课堂 / 课下）

---

## 4. 导师 Agent 的输出

建议也结构化：

```json
{
  "strategy": "guided",
  "focus": "auxiliary_line",
  "explanation": "如果连接这两点，可以构造新的相似三角形。",
  "next_question": "你先观察新增线段后出现了哪些角相等？"
}
```

---

## 5. 导师 Agent 的实现机制

核心流程：

### 第一步：识别学生意图

判断是“求答案”“问原因”“求提示”“纠错”。

### 第二步：定位问题来源

根据学生 Agent 判断：

* 是知识点没掌握
* 是能力不足
* 是当前这一步突然卡住
* 是表达不清但其实会

### 第三步：选教学策略

可以先写成规则系统：

```python
if mastery_low:
    strategy = "direct"
elif reasoning_weak:
    strategy = "guided"
else:
    strategy = "hint"
```

### 第四步：生成讲解内容

可分层输出：

* 结论层
* 原因层
* 方法层
* 迁移层

这样更像老师，而不是只给答案。

---

# 三、学生 Agent 与导师 Agent 的关系

它们不是并列独立的，而是前后依赖关系：

```text
学生行为/做题记录
        ↓
学生 Agent 建模
        ↓
得到学生当前状态
        ↓
导师 Agent 读取状态
        ↓
决定教学策略并生成讲解
```

也就是：

* **学生 Agent 决定“学生现在是什么样”**
* **导师 Agent 决定“接下来应该怎么教”**

# 四、最推荐的落地方式

先不要一开始做得太复杂，建议分三阶段。

## 第一阶段：规则驱动版本

先不用复杂模型，直接做可运行版本。

### 学生 Agent

* 用知识点掌握度表
* 用能力向量表
* 用规则预测卡点

### 导师 Agent

* 用意图识别
* 用规则选策略
* 用模板生成讲解

这个阶段最容易接进你现有系统。

---

## 第二阶段：数据驱动版本

等有了学生做题日志后再升级：

### 学生 Agent

* 根据历史正确率更新掌握度
* 根据耗时、卡顿、错因更新能力值

### 导师 Agent

* 根据历史交互数据优化讲解方式选择
* 逐步学会不同学生偏好的教学风格

---

## 第三阶段：个性化自适应版本

进一步做成真正的智能教学系统：

* 根据遗忘曲线动态衰减知识掌握度
* 根据学生近期表现动态调节讲解深浅
* 根据课堂/课下模式切换教学风格
* 根据连续提问轨迹判断是否需要回溯前置知识点
