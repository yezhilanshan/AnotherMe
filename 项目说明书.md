# AnotherMe Unified 项目说明书

版本：v1.0  
日期：2026-04-14  
适用范围：AnotherMe-V3 统一项目（AnotherMe + anotherme2_engine + API Gateway）

---

## 1. 项目简介

### 1.1 项目定位
AnotherMe Unified 是一个面向 AI 教育场景的一体化系统，目标是将“课程生成、互动课堂、题目讲解视频生成”打通为完整业务链路。

系统包含两条核心能力线：
1. AnotherMe 课堂线：主题/资料 -> 课堂大纲 -> 场景内容（Slides/Quiz/Interactive/PBL）-> 课堂回放。
2. AnotherMe2 视频线：题目图片/题干 -> 视觉解析 -> 脚本与语音 -> 动画生成 -> 讲题视频。

### 1.2 解决的问题
1. 降低课程内容生产成本：从人工备课转向自动化生成。
2. 提升互动体验：从静态课件转向多智能体课堂。
3. 统一异步任务：将长耗时生成任务纳入可观测、可重试、可追踪体系。
4. 支持多种部署方式：本地开发、Docker 编排、独立网关部署。

### 1.3 技术特点
1. 前端：Next.js 16 + React 19 + TypeScript。
2. 编排：FastAPI 网关 + 队列 Worker + 统一 Job 协议。
3. 引擎：LangGraph 多智能体工作流（especially 题目视频链路）。
4. 存储：PostgreSQL + Redis + MinIO/S3（可回退本地存储）。

## 3. 总体架构

## 3.1 分层架构
1. 表现层（AnotherMe Web）：
   - 提供课堂生成、预览、回放、题目视频发起等页面。
2. 业务与编排层（API Gateway）：
   - 提供统一任务 API。
   - 负责参数校验、入队、状态持久化、结果归档。
3. 执行层（Worker + Engines）：
   - AnotherMe 课堂生成执行。
   - AnotherMe2 题目视频执行。
4. 基础设施层：
   - PostgreSQL：任务元数据。
   - Redis：队列。
   - MinIO/S3：制品对象存储。

## 3.2 关键服务与端口
1. anotherme-core：3000。
2. api-gateway：8080。
3. PostgreSQL：5432。
4. Redis：6379。
5. MinIO API：9000。
6. MinIO Console：9001。

## 4. 业务流程说明

## 4.1 课程生成流程（course_generate）
1. 前端提交课程需求。
2. Gateway 校验 payload，生成 idempotency key，创建 Job（queued）。
3. Job 入队到 q.course。
4. Worker 消费任务，调用 AnotherMe 课程生成接口。
5. Worker 轮询 AnotherMe 任务状态，持续写入进度与事件。
6. 生成成功后写入 result（classroom_id/classroom_url/scenes_count）。
7. 前端查询任务结果并跳转课堂回放页面。

## 4.2 题目讲解视频流程（problem_video_generate）
1. 前端先上传题目图片到 Gateway 上传接口。
2. Gateway 返回 object_key 和 URL。
3. 前端创建问题视频任务，任务入队 q.problem_video。
4. Worker 执行 AnotherMe2 流水线：
   - Vision -> Script -> Voice -> Animation -> Repair -> Merge。
5. Worker 上传 final video 到对象存储并登记 JobArtifact。
6. 返回视频 URL、时长、步骤数、调试包 URL（如有）。

## 4.3 学习包流程（study_package_generate）
1. 支持 source.type=topic 或 photo。
2. 可选择输出 course/problem_video 的任意组合。
3. Worker 在一个 package job 中编排多个子任务并聚合结果。

## 5. 功能模块详细说明

## 5.1 AnotherMe Web 模块
1. 课堂生成：输入主题、语言、策略参数，触发课堂生成。
2. 生成预览：显示生成状态与中间过程。
3. 课堂回放：加载 stage + scenes 并进入沉浸式课堂页面。
4. 工作台页面：互动学习、计划、笔记等入口。
5. 题目视频页：上传图片、发起视频任务、查看执行结果。

## 5.2 API Gateway 模块（FastAPI）
入口文件：AnotherMe/anotherme2_engine/api_gateway/app.py

提供接口：
1. GET /：服务基础信息。
2. GET /healthz：健康检查（含队列可用状态）。
3. POST /v1/uploads：文件上传。
4. POST /v1/jobs：创建任务。
5. GET /v1/jobs/{job_id}：任务状态。
6. GET /v1/jobs/{job_id}/result：任务结果。

鉴权策略：
1. 当 GATEWAY_API_TOKEN 未配置时，接口可匿名调用（开发模式）。
2. 配置后要求 Authorization: Bearer <token>。

## 5.3 Job 与队列模块
1. 任务类型：
   - course_generate
   - problem_video_generate
   - study_package_generate
2. 任务状态：queued/running/succeeded/failed。
3. 队列名默认：
   - q.course
   - q.problem_video
   - q.package
4. 死信队列前缀：q.dlq。
5. 重试参数：max_retries=2，retry_base_seconds=5（默认）。

## 5.4 Worker 执行模块
入口文件：AnotherMe/anotherme2_engine/run_gateway_worker.py

机制：
1. 优先顺序消费队列：q.package -> q.problem_video -> q.course。
2. 支持 Redis 后端与 polling 后端。
3. 每次执行都会更新 Job 进度、step、事件。
4. 成功后写 result 和 artifacts，失败后写错误码与错误信息。

## 5.5 AnotherMe2 引擎模块
核心流程（题目视频）：
1. VisionAgent：题目识别、几何结构提取。
2. ScriptAgent：分步讲解脚本。
3. VoiceAgent：语音合成与时长对齐。
4. AnimationAgent：Manim 代码生成与布局管理。
5. RepairAgent：规则修复渲染错误。
6. MergeAgent：渲染与音视频合成。

## 6. 数据模型与协议

## 6.1 CreateJobRequest
```json
{
  "job_type": "course_generate",
  "payload": {},
  "user_id": "default_user"
}
```

## 6.2 各任务 payload 关键字段
1. course_generate：
   - requirement（必填）
   - language（默认 zh-CN）
   - options.enable_web_search/enable_tts 等
2. problem_video_generate：
   - image_object_key（必填）
   - problem_text（可选）
   - output_profile（默认 1080p）
3. study_package_generate：
   - source.type = topic/photo
   - outputs.course/problem_video（至少一个为 true）

## 6.3 JobSummary 返回体关键字段
1. job_id
2. job_type
3. status
4. progress
5. step
6. error_code/error_message
7. result
8. created_at/updated_at

## 7. 配置说明

## 7.1 根目录 .env.example（网关侧）
核心配置：
1. GATEWAY_HOST/GATEWAY_PORT：网关监听地址与端口。
2. GATEWAY_DATABASE_URL：PostgreSQL 连接串。
3. GATEWAY_REDIS_URL：Redis 连接串。
4. ANOTHERME_BASE_URL：AnotherMe 服务地址。
5. OBJECT_STORAGE_DRIVER：local/minio/s3。
6. OBJECT_STORAGE_*：对象存储参数。
7. GATEWAY_API_TOKEN：可选静态鉴权令牌。

## 7.2 AnotherMe/.env.example（应用侧）
核心配置：
1. LLM Provider：OPENAI/ANTHROPIC/GOOGLE/MINIMAX 等 API Key。
2. TTS Provider：TTS_OPENAI/TTS_MINIMAX 等。
3. ASR/PDF/Image/Video Provider。
4. DEFAULT_MODEL：后端默认模型。
5. ANOTHERME2_GATEWAY_BASE_URL/TOKEN：题目视频网关接入。

## 8. 部署与运行

## 8.1 本地开发（推荐）
前置条件：
1. Node.js >= 20.9。
2. pnpm >= 10。
3. Python 3.11（用于 Gateway/Worker）。

步骤：
1. 根目录复制 .env.example -> .env 并填写。
2. AnotherMe 目录复制 .env.example -> .env.local 并填写模型密钥。
3. 执行脚本启动：
```powershell
pwsh ./scripts/dev-up.ps1
```
4. 查看状态：
```powershell
pwsh ./scripts/dev-status.ps1
```
5. 停止服务：
```powershell
pwsh ./scripts/dev-down.ps1
```

运行日志：
1. .runtime/logs/anotherme-core.log
2. .runtime/logs/gateway.log
3. .runtime/logs/worker.log

## 8.2 Docker Compose 部署
```bash
docker compose -f docker-compose.unified.yml --env-file .env up -d --build
```
停止：
```bash
docker compose -f docker-compose.unified.yml down
```

容器组成：postgres、redis、minio、minio-init、anotherme-core、api-gateway、api-gateway-worker。

## 9. 运维与可观测性

## 9.1 健康检查
1. Gateway：GET /healthz。
2. AnotherMe：访问首页与关键 API。
3. Redis：由 Gateway healthz 透出 ping 状态。

## 9.2 任务监控
建议关注：
1. 任务总量与成功率。
2. 各任务平均耗时（按 job_type 分组）。
3. 失败原因 Top N（error_code）。
4. 队列积压长度。

## 9.3 常见故障处理
1. 网关创建任务失败：检查 .env 中 GATEWAY_DATABASE_URL、GATEWAY_REDIS_URL。
2. 任务长时间 queued：检查 worker 是否运行。
3. 课程任务失败：检查 ANOTHERME_BASE_URL 是否可达、AnotherMe API Key 是否配置。
4. 视频任务失败：检查对象存储配置和 AnotherMe2 依赖环境。

## 10. 安全与合规

## 10.1 接口安全
1. 生产环境必须启用 GATEWAY_API_TOKEN。
2. Gateway 建议仅暴露必要端口并置于反向代理后。
3. 对上传文件大小与类型增加策略限制（建议后续增强）。

## 10.2 数据安全
1. 对象存储建议按 job_id 分目录隔离。
2. 敏感配置仅通过环境变量注入，不写入前端。
3. 生产环境开启 HTTPS 与日志脱敏。

## 10.3 合规建议
1. 明确用户上传内容的数据保留期限。
2. 增加删除任务与制品清理机制。
3. 对 AI 生成内容增加人工复核流程（教育场景建议）。
